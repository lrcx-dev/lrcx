# LRCX 快速介绍

LRCX 是在传统 LRC 之上扩展出来的现代歌词文本格式。它保留了 LRC 最熟悉的时间轴写法 `[mm:ss.xxx]`，并精心设计了一套简洁易读的标记，将纯文本歌词、逐字时间、翻译、注音、声部、背景人声和自定义标记统一进同一个文件结构中。

如果已熟悉 LRC 规则，迁移到 LRCX 的核心过程只有两步：

1. 在文件开头增加 `HEAD` 声明区，用来声明元数据和可用标签。
2. 在 `BODY` 中继续写时间行，并按需追加翻译、注音、逐字时间等附加轨道。

## 目标

- 保持 LRC 作者可快速上手，不强迫改写原有时间轴习惯。
- 让“一个时间点对应多条附加信息”成为规范能力，而不是临时约定。
- 让解析器能稳定装配同一时间点的主歌词、翻译、注音、背景轨道和自定义标记。
- 让复杂歌词仍然可以回退到纯文本主行，允许渐进增强，保证人工维护成本可控。

## 核心特性

- `HEAD + BODY` 双区结构，`[---] v1.0` 作为版本分隔。
- 兼容基础 LRC 主行写法。
- 支持可选行持续时间：`[00:20.339+7945]`
- 支持逐字/逐词时间轨：`[#timing]`
- 支持翻译轨：`[#zh]`、`[#en]`、`[#trans]`
- 支持注音轨三种模式：`brief`、`full`、`supermark`
- 支持声部声明与行内声部标记：`[#voice:A,B,C]` + `[#A]`
- 支持背景人声轨：`[#back:offset_ms]`
- 支持引用前文行：`[#ref:time]`
- 支持自定义标签与标签属性：`[#marks:...]`、`[#OPTION:...]`


### 逐字时间轨 `#timing`

时间标记作用于其前面的文本片段：

```lrcx
[01:45.269][#timing]今<808>更<847>  <425>思<1001>い<601>出<184>す<425>な<574>よ<1096>
```

含义：

- `今` 持续 808ms
- `更` 持续 847ms
- 两个空格形成一个显式停顿片段，持续 425ms
- 后续字符继续按各自片段推进

### 翻译声明

```lrcx
[#trans:zh,en] 中文,English
```

- `zh`、`en` 变成可用的翻译轨标记：`[#zh]`、`[#en]`
- `[#trans]` 保留为“语言未指定”的通用后备写法，但不推荐作为规范写法。

### 注音声明

```lrcx
[#phonetic:pho1,romaji,romaji_full] brief,supermark,full
```

规则：

- 左侧声明具体注音轨名。
- 右侧按顺序声明每个注音轨的模式。
- v1 设计中支持三种模式：
  - `brief`
  - `supermark`
  - `full`

使用：
```lrcx
[00:20.339+7945][#pho1]so <240>ra <344>no <279>a <290>o <559>sa <567>ni <2496>me <274>wo <343>u <234>ba <639>wa <327>re <488>te <864>
```
上标模式：
```lrcx
[00:20.339+7945][#romaji]<^so ra>空^の<^a o>青^さに<^me>目^を<^u ba>奪^われて
```

### 声部声明

```lrcx
[#voice:A,B,C] ひとみ,まーしー,together
```

- `A`、`B`、`C` 是 `BODY` 中可用的具体声部标记。
- 右侧文本是给播放器或工具展示的人类可读名称。

使用：

```lrcx
[00:20.339+7945][#A]空の青さに目を奪われて
```

### 缓动声明

```lrcx
[#Easing:eo1,eo3] besier(0.25,0,0.75,1); besier(0.1,0.7,0.6,1)
```

- 用于高级逐字时间。
- 可使用高阶贝塞尔函数（上限7阶）。


示例：

```lrcx
[01:45.269+6384][#timing]今更<1912:eo1> <424>思い<1785:eo1>出すなよ<2263:eo3>
```

解释：

- 一个时间标记可覆盖多个字符
- `:eo1`、`:eo3` 使用 `HEAD` 中声明的缓动名
- `.halt` 用于表达等间距停顿；推荐只在确有需要时使用



## 与逐字LRC的区别

除了前文提到的功能性提升，设计思想上也有较大(激进)改进。
传统 LRC 的重点是“某一行在某个时间出现”。
LRCX 打破刻板约束，设计为“某一时间点对应一个歌词组，组内可以挂接多个轨道和标记”。
LRCX 简化了标记，从设计上将歌词功能增强，将歌词从“单行文本流”升级为“可装配的时间分组结构”。

标签系统增强了 LRCX 自定义功能，为后续拓展提供可能性。

## 文件骨架

```lrcx
[#title] 夏霞
[#artist] あたらよ
[#trans:zh,en] 中文,English
[#phonetic:pho1,romaji,romaji_full] brief,supermark,full
[#voice:A,B,C] ひとみ,まーしー,together
[---] v1.0

[00:20.339+7945][#A]空の青さに目を奪われて
[00:20.339+7945][#zh]被湛蓝的天空 夺走了所有的视线
[00:20.339+7945][#romaji]<^so ra>空^の<^a o>青^さに<^me>目^を<^u ba>奪^われて
[00:20.339+7945][#timing]空<584>の<280>青<848>さ<568>に<2496>目<273>を<343>奪<873>わ<328>れ<487>て<865>
```

## 从 LRC 迁移到 LRCX

### 原始 LRC

```lrc
[00:20.339]空の青さに目を奪われて
[00:29.593]足元の花に気付けないまま
```

### 对应的 LRCX

```lrcx
[#title] 夏霞
[#artist] あたらよ
[#trans:zh] 中文
[---] v1.0

[00:20.339][#A]空の青さに目を奪われて
[00:20.339][#zh]被湛蓝的天空 夺走了所有的视线
[00:29.593][#A]足元の花に気付けないまま
[00:29.593][#zh]仍未察觉到在脚边盛开的鲜花
```

由于LRCX能力几乎覆盖LRC，迁移无需一次用上所有能力。最常见的迁移路径是：

1. 先补 `HEAD` 元数据。
2. 再为已有主行补翻译轨。
3. 最后按需要补 `#timing` 或注音轨。

## LRC 与 LRCX 对比

| 能力 | 传统 LRC | LRCX |
| --- | --- | --- |
| 行级时间轴 | 支持 | 支持 |
| 行持续时间 | 非标准 | 标准支持 |
| 多语言翻译 | 依赖私约 | 标准支持 |
| 逐字时间 | 依赖私约 | 标准支持 |
| 注音 | 依赖私约 | 标准支持 |
| 声部 | 依赖私约 | 标准支持 |
| 背景人声 | 少见 | 标准支持 |
| 自定义标签 | 弱 | 标准支持 |
| 解析装配规则 | 不统一 | 有结构规则 |
