# LRCX v1 文本格式规范草案

本文档定义 LRCX v1 的规范写法。目标是让作者、转换工具和解析器对“文本应该长什么样”达成一致。

## 1. 基本结构

一个 LRCX 文件由三部分组成：

1. `HEAD`：元数据、标签声明、选项声明、注释。
2. 版本分隔行：`[---] v1.0`
3. `BODY`：按时间排序的歌词主体与附加轨道。

```lrcx
[#title] 夏霞
[#artist] あたらよ
[---] v1.0

[00:20.339]空の青さに目を奪われて
```

### 1.1 规范要求

- `HEAD` 必须出现在 `BODY` 之前。
- `BODY` 中使用到的系统标签或自定义标签，必须先在 `HEAD` 中声明，内建标签除外。
- 分隔行固定写作 `[---] v1.0`。
- 一个文件中只允许一个版本分隔行。

## 2. 行模型

LRCX 的通用行模型是：

```text
[LineTag+LineDuration][ContentTag][ContentTag]...ContentText
```

说明：

- `LineTag`
  - 在 `HEAD` 中是字符串键，例如 `#title`
  - 在 `BODY` 中是时间标签
- `LineDuration`
  - 可选
  - 单位为毫秒
  - 例：`[00:20.339+7945]`
- `ContentTag`
  - 可选，可重复
  - 用于标记本行的轨道类型、声部、自定义标签或属性
- `ContentText`
  - 该行的正文

## 3. BODY 时间标签

### 3.1 时间格式

`BODY` 时间标签的规范格式为：

```text
[hh:mm:ss.mmm]
[mm:ss.mmm]
[hh:mm:ss.mmm+duration_ms]
[mm:ss.mmm+duration_ms]
```

规则：

- 小时段可省略。
- 小时位数可为 1 位或更多位，不受 24 小时限制。
- 毫秒最多 3 位；少于 3 位时按右侧补零归一化。
- 超过 3 位毫秒精度应视为错误。

### 3.2 行持续时间

`+duration_ms` 表示整行的推荐持续时长。

```lrcx
[01:45.269+6384]今更 思い出すなよ
```

如果本行同时存在逐字时间轨：

- 行持续时间可以显式写在时间标签中。
- 也可以从整条时间轨求和得到。
- 两者同时存在时，推荐保持一致；解析器应将不一致视为语义错误。

## 4. 空白与转义

### 4.1 空白

- `ContentTag` 之间允许空格。
- `ContentText` 首尾允许空格，但作者应避免无意义空白。
- `HEAD` 中逗号分隔值时，逗号前后允许空格。

### 4.2 转义

反斜杠 `\` 用于把下一个字符按普通字符读入。  
建议至少支持以下字符的转义：

- `\[`
- `\]`
- `\<`
- `\>`
- `\^`
- `\\`

## 5. HEAD：元数据与声明

## 5.1 一等元数据字段

以下字段作为 v1 的标准内建元数据：

```lrcx
[#title] 夏霞
[#titlesort] Natsugasumi
[#artist] あたらよ
[#artistsort] Atarayo
[#album] 夏霞
[#lyricist] ひとみ
[#composer] ひとみ
[#date] 2021-08-11
[#releasetype] single
[#label] あたらよRecords
[#copyright] ℗ あたらよRecords
[#language] ja
```

附加字段可放入 `#more:*`：

```lrcx
[#more:isrc] JPPO02123456
```

## 5.2 贡献者与注释

```lrcx
[#by] loyuri
[#by:tran] translatorA
[#by:phonetic] annotatorA
[#note] Copyright reserved by original publisher.
[#note:by] annoymouse
```

## 5.3 声明标签是“模式生产者”

`HEAD` 中的声明行用于定义后续 `BODY` 可以使用哪些具体标记。

### 声部声明

```lrcx
[#voice:A,B,C] ひとみ,まーしー,together
```

- `A`、`B`、`C` 是 `BODY` 中可用的具体声部标记。
- 右侧文本是给播放器或工具展示的人类可读名称。

### 翻译声明

```lrcx
[#trans:zh,en] 中文,English
```

- `zh`、`en` 变成可用的翻译轨标记：`[#zh]`、`[#en]`
- `[#trans]` 保留为“语言未指定”的通用后备写法，但不推荐作为规范写法。

### 注音声明

```lrcx
[#phonetic:pho1,romaji,pho2] brief,supermark,full
```

规则：

- 左侧声明具体注音轨名。
- 右侧按顺序声明每个注音轨的模式。
- v1 设计中支持三种模式：
  - `brief`
  - `supermark`
  - `full`

### 自定义标签声明

```lrcx
[#marks:chorus,energy] Chorus section,Performance energy
[#option:energy.level,chorus.role] high,hook
```

规则：

- `#marks` 声明新的自定义标记名，右侧是标记注释。
- `#option` 为标签或标签属性提供默认值。这种方式下允许设定歌词默认值。
- 可以多次声明，结果取集合，后面的结果将覆盖前面的。
- 自定义标签用于补充结构信息，不改变本行的核心内容类型。

### 缓动声明

```lrcx
[#Easing:eo1,eo3] besier(0.25,0,0.75,1); besier(0.1,0.7,0.6,1)
```

- 用于高级逐字时间。
- 本草案延续现有关键字写法 `besier(...)` 以兼容既有设计。
- 可以多次使用`#Easing`声明缓动函数，默认取集合。同名缓动函数先声明的会被后声明的覆盖。

## 6. BODY：歌词组与轨道

## 6.1 同一时间点形成一个歌词组

同一个 `LineTag` 时间值的多行内容会被装配为同一个“歌词组”。

```lrcx
[00:20.339+7945][#A]空の青さに目を奪われて
[00:20.339+7945][#zh]被湛蓝的天空 夺走了所有的视线
[00:20.339+7945][#timing]空<584>の<280>青<848>さ<568>に<2496>目<273>を<343>奪<873>わ<328>れ<487>て<865>
```

这个歌词组里：

- 主行是首行文本行，`[#A]`是为主行增加的声部标记
- 附加了一条中文翻译
- 附加了一条逐字时间轨`#timing`行

## 6.2 BODY 中的标签是“模式消费者”

`HEAD` 只负责声明。`BODY` 中真正出现的 `[#A]`、`[#zh]`、`[#romaji]` 等才是被装配的具体标记。

### 主行可用形态

一个歌词组必须有且只应归一化为一个主行语义。主行可以由以下形态提供：

- 纯文本主行：`[time]Text`
- 带声部/自定义标签的主行：`[time][#A][#chorus]Text`
- `#timing` 主行：当没有纯文本主行时，可由时间轨还原文本
- `supermark` 注音主行：当没有纯文本主行时，可由 supermark 还原文本
- `#ref` 引用主行：引用前文已出现的主行内容

### 辅助轨道

默认命名推荐

- 翻译轨：`[#zh]`、`[#en]`、未知语言兜底`[#trans]`
- 注音轨：`[#romaji]`、`[#hiragana]`、`[#pho1]`、`[#pho2]`
<!-- - 背景人声轨：`[#back:offset_ms]`
- 注释忽略轨：`[#ignore]` -->

注：自定义标签只挂载在主行上

## 7. 逐字时间轨 `#timing`

## 7.1 基础语法

时间标记作用于其前面的文本片段：

```lrcx
[01:45.269][#timing]今<808>更<847>  <425>思<1001>い<601>出<184>す<425>な<574>よ<1096>
```

含义：

- `今` 持续 808ms
- `更` 持续 847ms
- 两个空格形成一个显式停顿片段，持续 425ms
- 后续字符继续按各自片段推进

### 推荐写法

推荐同时写上整行持续时间：

```lrcx
[01:45.269+5961][#timing]今<808>更<847>  <425>思<1001>い<601>出<184>す<425>な<574>よ
```

## 7.2 高级时间标记

LRCX v1 支持以下扩展形式：

- `<duration>`
- `<duration:easing>`
- `<duration:easing.halt>`
- `<duration:.halt>`

示例：

```lrcx
[01:45.269+6384][#timing]今更<1912:eo1> <424>思い<1785:eo1>出すなよ<2263:eo3>
```

解释：

- 一个时间标记可覆盖多个字符
- `:eo1`、`:eo3` 使用 `HEAD` 中声明的缓动名
- `.halt` 用于表达等间距停顿；推荐只在确有需要时使用

> 规范化建议：高级缓动与停顿属于进阶能力，核心作者流程仍应优先从纯文本主行和基础 `#timing` 开始。

## 8. 注音轨

## 8.1 `brief`

`brief` 用简洁文本表达注音，可附基础时长：

```lrcx
[00:20.339+7945][#pho1]so <240>ra <344>no <279>a <290>o <559>sa <567>ni <2496>me <274>wo <343>u <234>ba <639>wa <327>re <488>te <864>
```

适合：

- 罗马音
- 拼音
- 大多数对齐较规则的注音场景

## 8.2 `supermark`

`supermark` 通过复制主歌词文本并把注音片段包进 `<^...>` 与 `^` 中：

```lrcx
[00:20.339+7945][#romaji]<^so ra>空^の<^a o>青^さに<^me>目^を<^u ba>奪^われて
```

适合：

- 在可读性和结构绑定之间求平衡
- 让解析器直接恢复被标注的主歌词片段

## 8.3 `full`

`brief` 可以看作 `full` 的简化子集。
`full` 完全兼容 `brief` 标记，允许停顿标记和贝塞尔缓动：

```lrcx
[00:20.339+7945][#pho2]so<240> ra<344> no<279> a<290> o<559> sa<567> ni<2496> me<274> wo<343> u<234> ba<639> wa<327> re<488> te<864>
```

适合：

- 需要精细控制
- 需要缓动函数或更细的逐段控制

> 作为高级功能，单独声明将便于解析器快速检查支持情况。

## 9. 声部与背景人声

### 主声部

```lrcx
[00:20.339+7945][#A]空の青さに目を奪われて
```

- 未显式标声部时，默认采用 `#voice` 声明中的第一个声部。

### 背景人声

```lrcx
[02:09.719+8016][#back:2853]き<903>み<452>が<1545>
```

规则：

- `offset_ms` 表示相对当前歌词组起点的偏移
- 可为负数
- 背景人声轨附属于当前时间点的主歌词组

## 10. 引用前文行 `#ref`

```lrcx
[02:39.901+2032][#ref:02:37.502][#A]
```

规则：

- 只能顺序引用前文已经出现的歌词组
- 当前时间点仍然拥有自己的时间标签与可选持续时长
- 原有附加标签不继承；附加标签仍可写在引用行上
- 可复用附加轨可跟随引用，但实现时应保持来源可追踪；（翻译、注音、`#timing`轨可复用。`#back`与其他非标准附加行不可复用）


## 11. 推荐顺序

同一时间点的规范顺序建议如下：

1. 主行或引用行
2. `#timing`
3. 注音轨
4. 翻译轨
5. 背景人声轨
6. 其他补充行

这样做的好处是：

- 更接近人工阅读顺序
- 更利于 diff
- 更利于简单解析器流式装配
