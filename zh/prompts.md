
# 说明

本文件存放开箱即用的 LRCX 歌词增强/转换提示词，欢迎贡献！


## 为LRCX歌词补充上标注音（日语）

下面是全新歌词格式LRCX的简要介绍，阅读并遵循后续要求,必要的注释使用忽略行`[00.00.000][#ignore]`加内容在相应位置补充说明。

LRCX 文件由三部分组成：
1. `HEAD`：元数据、标签声明、选项声明、注释。
2. 版本分隔行：`[---] v1.0`
3. `BODY`：按时间排序的歌词主体与附加轨道。

`BODY` 时间标签的规范格式为：
```txt
[mm:ss.mmm+duration_ms]
```
`+duration_ms` 表示整行的推荐持续时长，可以省略。

声明标签是“模式生产者”。`HEAD` 中的声明行用于定义后续 `BODY` 可以使用哪些具体标记。

声部声明
```lrcx
[#voice:A,B,C] ひとみ,まーしー,together
```
- `A`、`B`、`C` 是 `BODY` 中可用的具体声部标记。
- 右侧文本是给播放器或工具展示的人类可读名称。

翻译声明
```lrcx
[#trans:zh,en] 中文,English
```
用法含义同上

注音声明
```lrcx
[#phonetic:pho1,romaji,pho2] brief,supermark,full
```
- 左侧声明具体注音轨名。
- 右侧按顺序声明每个注音轨的模式。
- v1 设计中支持三种模式：`brief`,`supermark`,`full`。需要上标注音选择`supermark`，需要缓动逐字支持选择`full`。

BODY：歌词组与轨道
同一时间点形成一个歌词组，应按行紧邻组装一起。
同一个时间值的多行内容会被装配为同一个“歌词组”。
```lrcx
[00:20.339+7945][#A]空の青さに目を奪われて
[00:20.339+7945][#zh]被湛蓝的天空 夺走了所有的视线
[00:20.339+7945][#timing]空<584>の<280>青<848>さ<568>に<2496>目<273>を<343>奪<873>わ<328>れ<487>て<865>
```
- 主行是文本行，`[#A]`是为主行增加的声部标记。
- 附加了一条中文翻译
- 附加了一条逐字时间轨`#timing`行，当没有纯文本主行时，可由`#timing`还原文本
- 时间标记后可附加多个以声明标记，比如：
`[00:20.339+7945][#A][#ignore] 这是一行注释`

时间标记作用于其前面的文本片段：
```lrcx
[01:45.269][#timing]今<808>更<847>  <425>思<1001>い<601>出<184>す<425>な<574>よ<1096>
```
- `今` 持续 808ms
- `更` 持续 847ms
- 两个空格形成一个显式停顿片段，持续 425ms
- 后续字符继续按各自片段推进

`brief` 用简洁文本表达注音，可附基础时长标记：
```lrcx
[00:20.339][#pho1]so <240>ra <344>no <279>a <290>o <559>sa <567>ni <2496>me <274>wo <343>u <234>ba <639>wa <327>re <488>te <864>
```

`supermark` 提供逐词上标音功能：
```lrcx
[00:20.339][#romaji]<^so ra>空^の<^a o>青^さに<^me>目^を<^u ba>奪^われて
```
- 上标注音片段以`<^...>`开头并记录上内容，以 `^` 结束，可跨越多个字
- `<^so ra>空^`中，`so ra`就是上标注音文本

主要Body轨道介绍完毕。补充：
- 常用标记：注释忽略轨`[mm:ss.xxx][#ignore]`
- 反斜杠 `\` 用于把下一个字符按普通字符读入。转换中歌词主体文本包含的`[,],<,>,^,\`应该被转义。

风格推荐：
- 不同时间行之间保留一个换行，相同时间行紧凑的排列在一起
- 总是遵循 纯文本、逐字时间行、注音(full、brief、romaji)、翻译的顺序
- 有`#timing`轨省略行时长

**任务说明**：
对于输入的LRCX日语歌词，为歌词的汉字增加上标注音标记。至少标注罗马注音上标（其次是假名上标），通常只标记汉字，连续汉字使用跨越标注。较长的罗马音标注可以增加空格划分音节。

选取假名读出罗马音遵循`supermark`组装上标注音行，可能需要在文件头部声明注音标签。
若歌词中文字后紧跟有被括号包裹的假名注释，可以删除并提取为假名上标注音。
除了`brief/full`的注音轨道，罗马音上标注音和假名上标注音轨道也可以同时存在，三者标签声明分别为 `pho1`,`romaji` 和 `hiragana`。

不要省略已有轨道，以纯文本输出增强后的完整LRCX歌词，不做解释。

<粘贴已有 LRCX>
