# LRCX 草案规范

LRCX 基于 LRC 改进，是功能全面、精简而对人类编辑友好的现代化歌词格式。

## 项目构成

本目录是 LRCX 的草案规范仓库，主要包含：

- `zh/`：中文规范草案
- `en/`：英文规范草案 (Pending)
- `showcase/`：示例文件

本仓库本身聚焦于格式设计、写法约定和解析规则草案。

阅读顺序参考：

1. 先读 [Readme.md](./zh/Readme.md) 了解整体设计。
2. 再读 [format.md](./zh/format.md) 了解文本格式。
3. 若需借助 LLM 工具，阅读 [convert.md](./zh/convert.md) 可能会有帮助。
4. 实例文件见 [lrcx-example.lrcx](./showcase/lrcx-example.lrcx)。

相关解析器实现位于项目仓库下，用于 LRCX 的解析与工程化使用。
已实现TS解析器 [LRCX-TS-Parser](https://github.com/lrcx-dev/lrcx-parser-ts)；


## 特性与快速预览

LRCX 在传统 LRC 的基础上扩展而来，保留熟悉的时间轴写法 `[mm:ss.xxx]`，并将纯文本歌词、逐字时间、翻译、注音、声部、背景人声和自定义标记统一进同一个文件结构中。

核心特性：

- `HEAD + BODY` 双区结构，`[---] v1.0` 作为版本分隔
- 兼容基础 LRC 主行写法
- 支持可选行持续时间：`[00:20.339+7945]`
- 支持逐字/逐词时间轨：`[#timing]`
- 支持翻译轨：`[#zh]`、`[#en]`、`[#trans]`
- 支持注音轨三种模式：`brief`、`full`、`supermark`
- 支持声部声明与行内声部标记：`[#voice:A,B,C]` + `[#A]`
- 支持背景人声轨：`[#back:offset_ms]`
- 支持引用前文行：`[#ref:time]`
- 支持自定义标签与标签属性：`[#marks:...]`、`[#OPTION:...]`

一个极简 LRCX 示例如下：

```lrcx
[#title] 夏霞
[#artist] あたらよ
[#trans:zh,en] 中文,English
[#phonetic:pho1,romaji,romaji_full] brief,supermark,full
[#voice:A,B,C] ひとみ,まーしー,together
[---] v1.0

[00:20.339+7945][#A]空の青さに目を奪われて
[00:20.339+7945][#zh]被湛蓝的天空 夺走了所有的视线
[00:20.339+7945][#romaji]<^so ra>空^の<^a o>青^さに<^me>目^を<^u ba>奪^われて
[00:20.339+7945][#timing]空<584>の<280>青<848>さ<568>に<2496>目<273>を<343>奪<873>わ<328>れ<487>て<865>
```

如果已经熟悉 LRC，迁移到 LRCX 的核心过程通常只有两步：

1. 在文件开头增加 `HEAD` 声明区，用于声明元数据和可用标签。
2. 在 `BODY` 中继续写时间行，并按需追加翻译、注音、逐字时间等附加轨道。


## 与逐字 LRC 对比

| 能力 | 传统 LRC | LRCX |
| --- | --- | --- |
| 行级时间轴 | 支持 | 支持 |
| 行持续时间 | 非标准 | 标准支持 |
| 多语言翻译 | 依赖私约 | 标准支持 |
| 逐字时间 | 依赖私约 | 标准支持 |
| 注音 | 依赖私约 | 标准支持 |
| 声部 | 依赖私约 | 标准支持 |
| 背景人声 | 少见 | 标准支持 |
| 自定义标签 | 弱 | 标准支持 |
| 解析装配规则 | 不统一 | 有结构规则 |
